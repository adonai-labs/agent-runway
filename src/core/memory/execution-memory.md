# Execution Memory

Operational memory for "do it right" signals.

Capture repeated delivery patterns:
- recurring implementation failures
- validation misses
- flaky test areas
- integration hotspots

## Entry Template

```markdown
## [YYYY-MM-DD] [pattern-title]
- Pattern type: [failure | success | safeguard]
- Where it happened: [module/path]
- Trigger: [what caused it]
- Consequence: [impact]
- Guardrail for next time: [specific check/rule]
- Linked artifacts: [spec/ticket/pr/test path]
```
